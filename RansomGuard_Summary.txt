----------------------------------------------------------------
3. FEATURES
----------------------------------------------------------------

3.1 CORE DETECTION ENGINE
--------------------------
- Real-time File Monitoring
  Detects mass renames, suspicious extensions (.locked, .encrypted),
  bulk file modifications in short time windows.

- Entropy Analysis
  Flags files with high encryption entropy score indicating
  ransomware encryption activity.

- Behavioral Analysis
  Monitors abnormal file I/O volume per process within time windows.

- Shadow Copy Protection
  Blocks VSS/backup deletion attempts via vssadmin or wbadmin commands.

- Process Monitoring
  Detects unsigned, injected, hollow, or suspicious spawned processes.
  Flags processes spawned from unusual parents (e.g. Word > PowerShell).

- Network Monitoring
  Flags C2 (Command & Control) communication attempts, unusual
  beaconing patterns, and DNS over HTTPS abuse.

- Honeypot Files
  Decoy files planted in common target directories.
  Triggers instant alert when any process accesses them.

- Registry Monitoring
  Watches for ransomware-related registry key changes
  and persistence mechanisms.

- YARA Rules Engine
  Signature-based detection for known ransomware families
  using SharpYara integration.

- Windows Event Log Parsing
  Parses Windows security audit logs for threat indicators.


3.2 RESPONSE & PREVENTION
--------------------------
- Auto Quarantine
  Moves suspicious files to an isolated quarantine folder automatically.

- Process Kill
  Automatically or manually terminate suspicious processes on detection.

- Network Isolation
  Cuts machine network connection on high-confidence ransomware detection.

- Memory Snapshot
  Captures process memory before termination for forensic analysis.

- Backup Protection
  Prevents unauthorized processes from accessing or deleting
  backup directories.


3.3 ALERTS & NOTIFICATIONS
----------------------------
- Real-time Alerts
  Native Windows toast notifications triggered immediately on detection.

- Severity Levels
  Four tiers: Critical / High / Medium / Low (color coded)

- Alert Details
  Each alert includes: process name, PID, file path, timestamp,
  threat type, action taken.

- Tamper-proof Logs
  Immutable SQLite audit trail of all events and actions taken.


3.4 REPORTS & EXPORT
----------------------
- Weekly/Monthly Reports   : Automated threat trend summaries
- Export Formats           : PDF and CSV
- Inline Charts            : Line chart (detections over time),
                             Bar chart (threat type breakdown)
- Date Range Picker        : Filter reports by custom date range

4. APPLICATION SCREENS
----------------------------------------------------------------
All screens share:
  - Narrow left icon sidebar (48-56px)
  - Top menu bar: File | View | Tools | Help
  - Bottom status bar: Protection status, Last Scan, DB Version, CPU%, RAM%
  - Segoe UI font, dark theme
  - Window optimized for 1280x800 resolution and above

SCREEN 1 — DASHBOARD (Main Screen)
  - Compact toolbar and narrow left icon sidebar
  - Small threat risk gauge/meter (not dominating the layout)
  - 4 compact stat rows: Files Monitored, Threats Detected,
    Processes Watched, Last Scan Time
  - 3 resizable panels:
      Left   : Real-time file activity log (scrolling rows)
      Center : Active threat list (tight table rows)
      Right  : System resource bars (CPU, RAM, Disk I/O)
  - Bottom status bar

SCREEN 2 — THREAT ALERTS
  - Full-width tight table:
    Severity | Threat Name | File Path | Process | Time | Action
  - Severity shown as small colored dot/badge
  - Right-click context menu: Quarantine, Kill Process, Whitelist, Details
  - Compact filter toolbar: dropdowns, search box, refresh button
  - Row count and filter status in status bar

SCREEN 3 — QUARANTINE MANAGER
  - Table: File Name | Original Path | Date | Threat Type | Risk | Size
  - Compact toolbar: Restore, Delete Permanently, Scan Again
  - Small storage usage bar at bottom
  - Right-click context menu on rows

SCREEN 4 — PROCESS MONITOR
  - Task Manager-style live process table
  - Columns: Name | PID | CPU% | RAM | File I/O | Network | Signature | Trust
  - Suspicious rows highlighted with subtle red row tint
  - Sortable columns with arrow indicators
  - Bottom toolbar: Kill Process | Whitelist | View Details | Refresh

SCREEN 5 — FILE ACTIVITY MONITOR
  - Sysinternals Process Monitor-style auto-scrolling log
  - Columns: Time | Event | Path | Process | Entropy | Risk Flag
  - Compact row height: 22-24px per row
  - Filter toolbar (one row): Event Type, Directory, Risk Level
  - Pause/Resume scroll button in toolbar
  - Row count in status bar

SCREEN 6 — REPORTS
  - Split layout: report list/history (left) | report preview (right)
  - Small inline charts (line + bar) — not full-page charts
  - Export toolbar: PDF, CSV, Print, Email Report (icon buttons)
  - Compact date range picker in toolbar

SCREEN 7 — SETTINGS
  - Two-panel layout: category list (left) | settings form (right)
  - Form style: label left, control right (like VS Options dialog)
  - Small native-looking toggle switches
  - Sensitivity: radio buttons (Low / Medium / High / Paranoid)
  - Exclusions: listbox with Add/Remove buttons
  - Apply and Cancel buttons at bottom right

SCREEN 8 — ONBOARDING WIZARD
  - Native Windows wizard style (like Windows Setup wizard)
  - Left panel: step list with current step highlighted
  - Step 1: Choose protection level
  - Step 2: Select directories to monitor
  - Step 3: Configure alerts and notifications
  - Back / Next / Finish buttons at bottom right

9. WHY WPF (DECISION RATIONALE)
----------------------------------------------------------------
CHOSEN: WPF + .NET 8 for v1

Reasons:
  + Deep native Windows API access (WMI, ETW, VSS, Registry, Event Log)
  + Best performance for system-level monitoring tasks
  + MSIX packaging — well supported on Microsoft Store
  + Mature, stable technology (.NET long-term support)
  + Rich .NET security libraries (SharpYara, BouncyCastle, etc.)
  + Native Windows look — trusted feel for small business users
  + Fastest path to market if team knows C#

Known Limitation:
  - Windows only (cross-platform requires rewrite in v2 with Tauri)

Decision: Ship fast on Windows → validate market → expand later.
